# Contributing

Thanks for your interest in contributing to No-BS-Sales-Enablement_V2!

## How to contribute

1. Fork the repository and create a feature branch from `main`.
2. Make changes in a topic branch with a clear commit message.
3. Add tests where applicable and run them locally.
4. Open a Pull Request describing the change and link any related issues.

## Code style
- Keep changes small and focused.
- Follow consistent formatting; add or update tests.

## Pull request process
- PRs must be reviewed by a code owner before merging.
- Rebase or merge `main` as needed to resolve conflicts.

If you have questions or want guidance, open an issue or DM @heath-gtm.
